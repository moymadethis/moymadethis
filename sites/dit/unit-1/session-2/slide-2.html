Audience needs

Think first

Audience needs can be determined by many different factors including age, income, interests, ability/previous experience and accessibility needs.

Let’s explore some of these key factors in more detail now – starting with accessibility needs.

The term ‘accessibility’ refers to the ease with which users can obtain, access and/or use an interface – different impairments can affect the way in which users are able to interact with an interface.

There are many different user accessibility needs to consider when creating a user interface. You can click through the slide panel below to explore these needs in more detail.

  • Visual impairment

    What is it?

    Someone with partial sight, limited vision or colour blindness.

    Design considerations

    • Use colours that contrast with each other to make sure a user can distinguish clearly between content
    • Build in functionality for text to be read aloud
    • Text size needs to be larger and with a clear simple font
    • Text size tool that allows text to be resized by the user

  • Hearing impairment

    What is it?

    Someone with partial hearing/deafness. Hearing might be muffled or limited to a narrow frequency range.

    Design considerations

    • Offer transcripts for podcasts/audio clips
    • Write text in a clear, easy to read language, perhaps incorporating graphics/images as well
    • Use subtitles for video content

  • Speech impairment

    What is it?

    Someone who struggles to communicate verbally. This might be a stutter, stammer, lisp or other impairment.

    Design considerations

    • Provide non-speech methods for interaction such as entering characters via touchscreen/keyboard
    • Provide sufficient time and multiple attempts for speech interface use

  • Physical (motor) impairments

    What is it?

    Someone who struggles to use their body in a fluid manner conducive to using standard interface inputs. This could be limited movement/control in limbs or limbs missing entirely.

    Design considerations

    • Provide alternative methods for interaction including voice/speech commands and visual user interfaces
    • Provide shortcuts for different tasks
    • Make navigation and other key features larger and more prominent to enable a user to easily find and select them

  • Cognitive impairment

    What is it?

    Someone who needs longer processing time, simplistic wording and clear navigation.

    Design considerations

    • Navigation and content is laid out clearly and consistently
    • Speech interface is offered as an alternative to written commands
    • Spell check feature is considered for input boxes
